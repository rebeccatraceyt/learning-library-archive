The npx-create-react-app command

The npx create-react-app command is used to create a new React application quickly and easily.
It's a command-line tool that will generate the boilerplate code and project structure for a new React application.
This includes setting up the basic file structure, configuring build tools, and installing essential dependencies.
This allows you to start coding your React app straight away.

The full command is:

user@ci-os /home/user (main) $ npx create-react-app [app_name] --template [template_name] --use-npm   

Now, let's break this down:

* Note: Flashcards are optimised for larger screens due to their compact design.

Mandatory Commands

This mandatory command tells npx (node package execute), which allows us to run packages without installing them, to use the create-react-app package.
npx create-react-app [app_name]
The [app_name] is the name that you want to give your application.
All react app names must be in lowercase letters. If you are already in a folder that you want the application to be created in, you can simply use a dot [ . ] in place of your application name.
npx create-react-app .
Or
npx create-react-app my-app-name

Optional Commands

The other half of the command contains 2 optional flags:
A template name/URL to use when installing the application. We have used one here in order to keep our React version the same as the version we will use in the course material.
--template [template_name]
The default package manager to use. By default the Yarn package manager is used unless you specify another (such as npm, which will be our choice in this application).
--use-npm
