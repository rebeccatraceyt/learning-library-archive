Stripe Webhooks

Stripe Webhooks are like real-time notifications from the Stripe payment system to your application.
They let your code know when important events, such as successful payments or subscription changes, happen.
By using webhooks, you can automatically update your app or perform specific actions based on these events, making your payment processing smoother and more efficient.
Just like any other type of functionality, Stripe Webhooks have the potential to break. This flashcard offers a detailed guide on where to start troubleshooting (and the likely cause), when that happens:

* Note: Flashcards are optimised for larger screens due to their compact design.

Error: 400

Status
400
Symptoms
Bad request on checkout
Cause
Environment variables
Solution
Check your keys are accessible
(learn how to do that here)

Error: 401 or 403

Status
401 or 403
Symptoms
Webhook URL (checkout/wh/) not accessed in the terminal
Cause
Server port is not public
Solution
Set local server port to public
(learn how to do that here)

Error: 404

Status
404
Symptoms
Webhook URL (checkout/wh/) not accessed in the terminal
Cause
Incorrect endpoint URL
Solution
Check endpoint is using updated local server URL
(learn how to do that here)

Error: 500

Status
500
Symptoms
Webhook URL (checkout/wh/) not accessed in the terminal
Cause
An error in your code
Solution
Reading the error may give indication on how to debug
(learn how to do that here)

Error: 500

Status
500
Symptoms
Duplicate Orders
Cause
Existing order cannot be found in webhook_handler.py due to inconsistent address, grand_total, etc
Solution
Ensure order details do not change after the intent is created on the checkout page

Error: 500

Status
500
Symptoms
Heroku-only duplicate Orders
Cause
Signals for OrderLineItem updates are not unique
Solution
Give signals unique names
