Python List Comprehension

What is it?

List comprehension is a Pythonic method of creating lists. List comprehensions allow us to efficiently create lists in fewer lines of code than other methods. A major benefit of list comprehension is that it can be used to create and filter lists in just a single line of code.

Now we know what the concept is, how do we go about using it? Let’s dive in and take a look!

names = ['Barry, 'Anika', 'Zuri', 'Tomas']
new_names_list = []

for name in names:
if name[0] != 'T':
new_names_list.append(name)

print(new_names_list)

All we’re doing is running through the list of names, checking if the first letter is ‘T’ and, if it isn’t, adding it to the new names list. As expected, we get the same list of names, just without Tomas (sorry, Tomas).

['Barry, 'Anika', 'Zuri']

So, how would we approach this with a list comprehension? We know the basic structure from our previous ‘names’ example, but where would the conditional statement fit in? The list comprehension solution looks like this:

names = ['Barry, 'Anika', 'Zuri', 'Tomas']

new_names_list = [ name for name in names if name[0] != 'T']

print(new_names_list)

The conditional statement comes after the name of the list that will be iterated. This takes the form:

for member in iterable if member meets condition

We took a ‘for’ loop that was three lines in length and condensed it down into a single line of code. Pretty neat!

There is one important caveat to cover here. Our ‘if’ statement appeared at the end of the comprehension because we simply wanted to check if a value equalled something (a ‘T’ in this case). What would we do if we wanted to change a value in the list (rather than just omit it) based on whether a certain condition was met? The answer – we’d put the ‘if/else’ condition immediately after the expression instead:

numbers = ['1, '22', '5', '65', '23', '42', 8]

new_names_list = [ i if i in names if name[0] != 'T']

Relative paths

Relative paths are relative to the current directory. If we refer to the project structure above, the relative path from mysite/ to style.css is:

./assets/css/style.css

In our HTML file, this would be:

<link rel=”stylesheet” href=”./assets/css/styles.css”>

Note that we start the path with ./ This indicates that the path starts in the current directory. Omitting this would cause the CSS file not to load on certain servers.

If we want to load our style.css file into contact.html, we can use the same absolute path as earlier. The absolute path has not changed:

/mysite/assets/css/style.css

We can import it using the same code as before:

<link rel=”stylesheet” href=”/assets/css/styles.css”>

However, the relative path is different, as the path is now relative to a different directory (contact, in this case).

The relative path from contact/ to style.css is:

../assets/css/style.css

Note that we now start with two full stops: ../

This indicates that we should go up one level in our directory structure. In this case, this will move us up to the mysite/ directory. From there, we can traverse back down into assets, then css, then style.css.

We can also use this syntax in our HTML file:

<link rel=”stylesheet” href=”../assets/css/styles.css”>
