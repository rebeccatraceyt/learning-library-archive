CSS Box Model

The CSS Box Model is appropriately named. Every single webpage is essentially built using 2D boxes - a lot of 2D boxes. Boxes sat next to each other, stacked on top of each other, nested inside of each other, and so on.
Just like physical boxes, CSS boxes have dimensions which are important when you are trying to fit them all together. Most layout problems are caused by incorrect measurements, making the boxes fall out of alignment or not fit inside other boxes. Understanding how to adjust your box measurements and what tactics you can use to avoid overlaps is therefore very important.

What is it?

If you’ve spent any time in your browser’s Dev Tools, you’ll have come across the image below, which is a representation of a particular element’s box. It shows you the space taken up by the actual element (in the center), then its padding, then its border, and finally its margin.

Let’s look at an example:

Below on the left is an illustrated HTML element, with the red dashed border representing the edge of the margin. Underneath thatOn the right, we have the style rule that is applied to it:

.box {
height: 150px;
width: 150px;
padding: 32px;
border: 2px solid;
margin: 32px;
background: pink;
}

Important things to note here:

Background properties only apply to the area taken up by the content and padding
The ultimate dimensions of the element are larger than the height and width specified in the box’s styling due to the padding and border

How do we use it?

Historically, you would need to perform some calculations to work out the total dimensions of your box. In the example above, to find the total width or height we would need to add the margins, border, and padding to the content height:

32 (margin)
2 (border)
32 (padding)
150 (content)
32 (padding)
2 (border)
32 (margin)

282px

The fact that an element may have any or all of these additional sizes applied is a common cause of unexpected layout results, particular overflows. Thankfully, you can simplify things a little by changing the box-sizing value of an element:

box-sizing: border-box;

By changing the box-sizing value of your elements, any height and width properties you set will include the border and padding. Applying this to our example above means that the whole width of our box is now 150px (excluding 64px for the margin), rather than 282px. This property is commonly applied to all elements in a document. Dev Tools would now show this box model for our element:

We have hard-coded values for the width and height of our box (150px by 150px). In practice, this is rarely the best approach as it fails to take into account the size or aspect ratio of the content inside it and blindly follows your measurements. This is what will happen when we add more content to our existing box:

The maximum area available in our box will be 22,500px (150 x 150), which is not big enough to fit all of our content. If we simply remove the height property, the browser will increase our box’s height dynamically, and our box will grow to fit the content while keeping its width of 150px:

For responsive design

Rather than rely on finding ‘magic numbers’ to size our boxes, it would be easier if we could use percentages or some other responsive unit of measurement to make things fit to various screen sizes. Fortunately, CSS allows us to do just that!

Using percentages to set the height or width will depend on the height and/or width of the parent element that contains our box. Setting a width of 100% on our box will make it as wide as its container. Consider the following set up:

* {
box-sizing: border-box;
}
.box {
width: 100%;
padding: 32px;
border: 2px solid;
}

This box will now stretch to be as wide as its container.

For default resets

Another important thing to be aware of is that some HTML elements have margin and paddings applied to them by default. To give yourself the best chance of avoiding the pitfalls of box sizes, you should always set the default margin and padding to zero in your universal selector at the top of your CSS:

* {
margin: 0;
padding: 0;
box-sizing: border-box;
}

You can now be confident that only the margins and paddings that you have specified will be applied to the elements in your HTML document.

Helpful hint!

Sometimes you will want to add borders to elements to give the user some visual feedback, e.g. when hovering over links in your navbar:

.nav-link:hover {
border: 3px solid white;
}

You may notice that when the border appears it shifts things slightly, but knowing that borders occupy space explains this subtle change.

A neat trick to avoid this issue is to use the outline property, which does not take up any space in the page and therefore does not cause your other elements to move about:

.nav-link:hover {
outline: 3px solid white;
}
