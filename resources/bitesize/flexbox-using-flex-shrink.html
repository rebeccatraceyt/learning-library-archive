Flexbox

Using flex-shrink

The flex-shrink property works in much the same way as flex-grow, except it works in reverse – the number you set on the flex item determines how much the item is able to shrink, relative to other flex items.

We’ll need to add a few more items to our example this time this time, as we’re trying to take less container space rather than more.

If we set our code to the following:

<div class=”flex-container”>
<div>1</div>
<div>1</div>
<div style="flex-shrink: 5;">3</div>
<div>4</div>
<div>5</div>
<div>6</div>
<div>7</div>
<div>8</div>
<div>9</div>
<div>10</div>
</div>

We would expect this code to cause the element numbered 3 to be smaller than the others, as it is being told to shrink more than the other elements (which don’t have a flex-shrink property set at all, and therefore default to 1). Let’s check the output:

Sure enough, element 3 has shrunk in size more than the others, due to having a higher flex-shrink value set. We could of course mix and match some values across the six elements, to get a better idea of what the flex-shrink property can do - in this case we have applied flex-shrink values to elements 3, 5 & 6. While achieving this type of sizing wouldn’t be too much of an issue with regular CSS, getting all the elements to play ball across different screen sizes definitely would be. The flexbox model provides an easy solution to this problem!
