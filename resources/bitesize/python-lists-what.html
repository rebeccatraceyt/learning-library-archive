Python List Comprehension

What is it?

List comprehension is a Pythonic method of creating lists. List comprehensions allow us to efficiently create lists in fewer lines of code than other methods. A major benefit of list comprehension is that it can be used to create and filter lists in just a single line of code.

Now we know what the concept is, how do we go about using it? Let’s dive in and take a look!

Let’s suppose we have a list of names, and for whatever reason, we want to copy the names to a new list. A naive solution to this problem would would be to use a standard ‘for’ loop which runs through the list of names and appends them to a new list:

names = ['Barry, 'Anika', 'Zuri', 'Tomas']
new_names_list = []

for name in names:
new_names_list.append(name)

print(new_names_list)

This will result in the below output:

['Barry, 'Anika', 'Zuri', 'Tomas']

The output is correct, so you may be wondering what’s wrong with the above code? Well, there’s nothing technically wrong with it. It does the job just fine.

However, it doesn’t leverage all of the language features Python makes available to us. Python emphasises less-cluttered syntax, which allows us to do more with less code, while maintaining readability. This style of coding is referred as Pythonic, and Pythonic code is sometimes described as being “beautiful.”

So, let’s put that claim to the test! Take a look at the following code, which achieves the same result as our first example:

names = ['Barry, 'Anika', 'Zuri', 'Tomas']

new_names_list = [name for name in names]

print(new_names_list)

Our result is the exact same:

['Barry, 'Anika', 'Zuri', 'Tomas']

We’ve managed to take the previous for-loop approach and turn it into a single line of code, which does the exact same thing. Beautiful, no?

The structure of a list comprehension looks like this:

[expression for member in iterable]

When starting out, it is generally easier to read the expressions within a list comprehension from right to left. In the above, let’s start with:
for member in iterable (this maps to for name in names in our concrete example above).
This is a simple for loop, and tells Python to loop through the members in our list.

The expression (the first instance of name in the concrete example), is what will be put into our new list. By saying name for name, we are telling Python that we want the names copied exactly as they are.

In other words, the code [name for name in names] tells Python to loop through the names list, and create a new list out of each name within it.

This can be a bit confusing to get your head around at first, but as we look at more examples in the next few cards, it will become clearer.
