Navbars

The navbar is the core navigation point for users of your website. Just as you would have traditional signage in a brick and mortar shop, the navbar is the signposting for your website, enabling users to quickly find and navigate to whatever resource they are looking for. Your navbar should be easily accessible on every page of your website, allowing visitors to quickly and easily change the direction of their user journey if required.

When creating a navbar, UX must be at the core of the design

With this in mind, when wireframing the navigation for your site, you should have your list of user stories to hand in order to guide you in the right direction:

“As a user I want to be able to navigate easily so that I can find sandals, socks, trainers or boots”


Elements & Structure

In HTML, the navigation <nav> element is generally placed inside the <header> element. At the very least, it should contain a list of links to the important pages in your site. These links can then be styled with CSS to make the navbar fit in with the overall style of your site. Here is a very simple map of the element structure:

<header>
<nav>
<ul>
<li>
<a href='#'>
Item 1
</a>
</li>
<li>
<a href='#'>
Item 2
</a>
</li>
</ul>
</nav>
</header>

For a basic working example using HTML, CSS & JavaScript take a look at our Simple Navbar example on CodePen!

Keep it simple, semantic, and SEO-friendly!

Do keep Nav items to a max of 5
Do use semantic, descriptive link names
Do use Aria labels for accessibility
Do ensure to include a mobile nav / hamburger
Do ensure to include a home link with logo in the top left
Use active classes to highlight the active page
Give interactive feedback (hover, pointer-cursor)
Don't overcrowd your Navbar
Don't use generic link names like 'products' & 'services'

Semantic, relevant, accessible menu options will result in better click-through as well as better SEO
