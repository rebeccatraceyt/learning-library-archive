React Iteration

(loops)

Maps

When using React, JavaScript’s Array.prototype.map is the most common type of loop you will encounter, and is the one you will use 99% of the time. It is generally used to iterate over the indices of an array and return a new array of JSX, one for each index. Let's inspect how “map” actually works in React before going any further:

Map will take an existing array, and loop over each index

The map function itself takes a callback function as an argument

const myArray = ['Seán', 'John', 'Rebecca', 'Ger'];
const newArray = myArray.map(() => {}); // loops over each index

Map takes 3 parameters which are implicitly passed to the callback function:

All three parameters are optional, but usually at least the first is needed.

1 The element at the current index in the loop.
2 The current index in the loop (an integer).
3 The entire array that the map function is looping over.
const myArray = ['Seán', 'John', 'Rebecca', 'Ger',];
const newArray = myArray.map((element, index, array) => {});

Map requires a return value for each iteration

In this case, we will return a paragraph for each index. When complete, a new array of JSX will be stored in the newArray variable.

const newArray = myArray.map((element, index) => {
return <p key={`names-${index}`}> {element} is at index {index} </p>
});

The key Prop

You will notice the “key” property on the <p> element in the above example. A unique key is required for the outermost JSX element.

React uses the key prop to create a relationship between the component and the DOM element. It uses this relationship to determine whether or not the component should be re-rendered.

In this example, we will give it a custom name and an index number. Assigning only the index number is not enough in most cases, as you may have other DOM elements with duplicate keys. If this happens, React will not know which component to re-render.

<p key={`names-${index}`}>

Adding the array to the JSX

You can add an array of JSX to a React component in a few different ways:

As a variable:

const newArray = myArray.map((element, index) => {
return <p key={`names-${index}`}> {element} is at index {index} </p>
});

return (
<>
{ newArray }
<>
);

Directly in the JSX:

return (
<>
{
myArray.map((element, index) => {
return <p key={`names-${index}`}> {element} is at index {index} </p>
});
}
<>
);

Or even as a function call:

const myFunc = () => {
const myArray = ['Seán', 'John', 'Rebecca', 'Ger'];
return myArray.map((element, index) => {
return <p key={`myKeyName-${index}`}> {element} is at index {index} </p>
});
};

return (
<>
{ myFunc() }
<>
);

Iterating an object

What if you have an object instead of an array? You may have a very large object that was returned from a database or an API. You may not want to type out a lot of JSX when you can do it in just a few lines of code.

We will still use map, but since map is a method of the Array prototype, we first need to convert all the keys of the object to an array and map over them.

Here we are iterating over a small object, but the same principle applies to larger objects:

The example code:

export const Loops = () => {
const myFunc = () => {
1 const myObject = { name: 'Seán', age: '99', eyes: 'Blue' };

2 return Object.keys(myObject).map((key, index) => {
return <p key={index}> {key}: {myObject[key]} </p>
});
};

return (
<>
3 { myFunc() }
<>
);
};

The example output:

4

name: Sean

age: 99

eyes: Blue

The example explained:

1 We create the object we will iterate over.
2 We create an array of the object’s keys using the Object.keys() method, then using method chaining we loop/map over the keys and create a new array of JSX to display.
3 We call the function to return the array which will be spread into the JSX.
4 The result is displayed on the webpage.

Iterating an array of objects

We can use the map method once again to iterate an array of objects. The currently iterated object will be passed as a parameter. In the below example, it is called “ object” for brevity.

We use dot syntax to access the key values in the object.

The example code:

export const Loops = () => {
1 const myArrayOfObjects = [
{ name: 'Seán', age: '99' },
{ name: 'Ger', age: '98' }
];
2 const myFunc = () => {
myArrayOfObjects.map((object, index) => {
return <p> {object.name} is {object.age} </p>
});
};

3 return (
<>
{ myFunc() }
<>
);
};

The example output:

4

Sean is 99

Ger is 98

The example explained:

1 We create the array that will contain the objects we'll need.
2 We use JSX to return each array object. Dot syntax is used to access the object's key values.
3 We call the function to return the array of objects.
4 The result is displayed on the webpage.

Adding items to an array in useState

Each update to useState must be a new object. As such, in order to push a new item into an array, we can use the spread operator. We can spread the original array into the new array, followed by any other values we want to add.

const [myArray, setMyArray] = useState([]);
setMyArray([...myArray, 'Item to be added to myArray']);

There are times in React where you may need to use a callback function. The old state value will be implicitly passed to the callback function as an argument.

const [myArray, setMyArray] = useState([]);
setMyArray(oldArray => [...oldArray, 'Item to be added']);

As an example, let's see how we could use this in a component.

We will set an empty array to the useState, and assign a click handler to a button.

Each time the button is clicked, the setMyArray update method will set a new array containing the old array values and any new values you wish to add.

When the state has updated, React will re-render the component which will cause the myFunc function to run again, this time displaying the update array items.

The example code:

import { useState } from 'react';
export const Loops = () => {
1 const [myArray, setMyArray] = useState([]);
2 const handleClick = () => {
setMyArray([...myArray, `${myArray.length + 1} Clicks`]);
};
3 const myFunc = () => {
myArray.map(element => <p key={element}> {element} </p>);
};

4 return (
<>
<button onClick={handleClick}> Increase Clicks </button>
{ myFunc() }
<>
);
};

The example output:

5

1 Clicks

2 Clicks

3 Clicks

The example explained:

1 We create the array and the array update method for the useState Hook.
2 We create the click handler, which sets a new array to state on each triggered event (when the button is clicked).
3 We use the function to render an array of JSX elements.
4 We use JSX to create a button and inject our handler. We return the function to render each object of the array.
5 The result is displayed on the webpage.

Removing items from an array in useState

Removing an item is a little more tricky, but the Array.prototype.filter method is a great use case.

Filter, like map, returns a new array which is needed for React state changes.

Given a list of names, let's remove the string “Sean” from the array example below:

const [myArray, setMyArray] = useState(['Seán', 'Rebecca', 'Ger']);
setMyArray(myArray.filter((element, index) => element !== 'Seán'));

When React notices the state change, it will re-render your component.

Adding items to an object in useState

Much like an array, we also need to provide a new object to useState. The click handler will set the state by spreading the old object values and any additional key/value pairs into a new object.

The example code:

export const Loops = () => {
const [myObject, setMyObject] = useState([]);
1 const handleClick = () => {
setMyObject({...myObject, name: 'Seán'});
};
2 const myFunc = () => {
Object.keys(myObject).maps(key => <p key={key}> {myObject[key]} </p>);
};

3 return (
<>
<button onClick={handleClick}> Add Name </button>
{ myFunc() }
<>
);
};

The example output:

4

Seán

The example explained:

1 We create the click handler, which will set a new object into state
2 The function will render an array of JSX elements
3 We use JSX to create a button and inject our handler. We return the function to render each object of the array.
4 The result is displayed on the webpage.

Removing items from an object in useState

To remove items from an object, we can simply make a copy of the object in state, delete the key, and return the updated copy.

const [myObject, setMyObject] = useState({name: 'Seán', age: '99'});
const handleClick = () => {
let newObject = {...myObject};
delete newObject['name'];
setMyObject(newObject);
};

We can also destructure the old object by separating the key to remove from the rest of the object.

const handleClick = () => {
setMyObject(oldObject => {
const{name, ...remaining} = oldObject;
return remaining;
});
}

Updating keys in an object in useState

Updating a key’s value is much the same as adding a new key, with the slight difference that the key to update already exists. Therefore, its value will be overwritten.
