Dynamic Debug

When working on your project, you will want to dynamically change your DEBUG setting without modifying your code. To achieve this, we can make the DEBUG setting dependent on the presence of an environment variable called DEVELOPMENT, which will enable debug mode for us if present.

Think about a dynamic debug value as a switch you toggle on and off.

You do this with a variable called: DEVELOPMENT

Let's look at how to do that:

1. In settings.py

Replace your current DEBUG variable with:

DEBUG = 'DEVELOPMENT' in os.environ

This line of code will conditionally set the DEBUG variable, depending on the existence of a DEVELOPMENT variable in your env.py file.

The condition we are setting here is:

If a DEVELOPMENT variable exists in your env.py file, DEBUG will be True for your local development environment

2. In env.py

Create a variable named DEVELOPMENT and set its value to 1

os.environ["DEVELOPMENT"] = "1"

DEBUG is now switched on correctly for your local environment.

To switch DEBUG off for your local environment, simply remove this variable (and its value) from your env.py file.

3. In your deployment’s environment variables

While you are working on your project and debugging you can add the DEVELOPMENT variable into your environment variables

You can set the DEVELOPMENT variable with:

DEVELOPMENT = "1"

Remember: On submission there should be no DEVELOPMENT variable in your deployment’s environment variables
