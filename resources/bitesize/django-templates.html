Django Templates

(explained)

What are templates?

A Django template is a HTML file which contains the structure of the pages we want our users to see.

Django uses its own template language, called the Django Template Language (DTL).

As we are working with a database in Django, we usually need our website to display data dynamically from the database. An example of this would be a product page in an ecommerce site - we only want to write one HTML file which contains the structure (our template), which will then be used to display many different products.

DTL allows us to do this, as we can add variables and logic inside our HTML files, inside defined blocks. This is not possible with standard HTML.

How do we use template blocks?

Templates use blocks of code, which greatly cuts down on the amount of HTML code we have to write.

Blocks are easy to use and offer great flexibility. Here's how to use them:

Create a base template (base.html) which contains the general structure/layout of your website (e.g. navbar, footer), but no specific content.

In your base.html, add a content block:

{% block content %}
{% endblock %}

When creating a new page (e.g. products.html), you can now extend base.html, which will essentially make this file inherit its code. Write any custom code for this particular page into the content block:

{% extends "base.html" %}
{% block content %}
<div>
Content goes here
</div>
{% endblock %}

You can name your blocks anything, as long as the names across multiple files are consistent. Give them meaningful names based on how they are used.

Now when your products.html template renders, it will be included inside the base.html file (i.e. the content block from products.html will replace the empty content block in base.html). This will give us a single rendered HTML page.
