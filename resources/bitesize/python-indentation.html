Python Indentation

Unlike JavaScript, Python doesn't use brackets to specify where blocks of code (e.g. function bodies, conditional blocks, and loops) begin and end. It uses indentation (using four spaces, to comply with PEP 8).

Let’s have a look at some examples:

Function blocks:

1 def my_function():
2 num = 2
3 str = f"number is equal to{num}"
4 return str
The colon at the end of line 1 indicates the start of an indented block
Lines 2 and 3 are part of the function as they have 4 spaces before them (i.e. they are indented)

If statements:

1 if num < 10:
2 print("number is less than 10")
3 elif num > 10:
4 print("number is greater than 10")
5 else:
6 print("number is equal to 10")
7 print("this line is not indented!")
The code on line 2 is indented, so it is part of the if block (line 1)
The same applies to the elif and else blocks
Line 7 is not indented, so it is not part of the else block

Here’s an example of incorrectly indented if statement:

1 if num < 10:
2 print("number is less than 10")
3 elif num > 10:
4 print("number is greater than 10")
The elif should not be indented here. It needs to be vertically aligned with the if statement on line 1

Nested Statements:

For nested statements you indent by another 4 spaces:

1 if num < 10:
2 print("number is less than 10")
3 if num > 5:
4 print("number is greater than 5")
5 if num == 6:
6 print("number is equal to 6")
7 elif num == 7:
7 print("number is equal to 7")
Each time you nest a statement, you indent the nested code block.

If your code is not working as expected, a good debugging tip is to use a Python validator to ensure there are no indentation errors. You can use the Code Institute Python Validator to do this!
